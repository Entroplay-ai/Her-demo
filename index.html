Required

Your backend

Her does not talk directly to OpenClaw from this static demo repo. You need your own her-bridge instance in front of your own OpenClaw Gateway, configured by whoever runs your backend.

  • Run npx tsx src/index.ts setup in her-bridge/
  • Enter your OpenClaw gateway URL and token
  • Start the bridge and copy the printed tunnel URL
  • Paste that link into web or macOS to pair

How pairing works

Set up the bridge once, then use the same link everywhere.

1. Set up her-bridge

On the machine that can reach OpenClaw, give the backend owner her-bridge/, run npm install, then run npx tsx src/index.ts setup. Enter the OpenClaw gateway URL and token. If you want remote web/macOS pairing, also enter the tunnel API URL and invite code.

2. Start the bridge

Run npx tsx src/index.ts start. When the bridge is ready, it prints a Tunnel URL: wss://... line. That is the value you use for pairing.

3. Pair web or macOS

Open the web demo or the macOS app. When Connect to Bridge appears, paste the wss://... tunnel URL. After the status changes to Connected, voice, text, and video mode will use your OpenClaw backend through her-bridge.

Before users click

A few things to know before you start.

macOS build note

The current macOS demo is distributed through GitHub Releases. If it is unsigned or not notarized, macOS can block the first launch until you use Open Anyway in Privacy & Security.

Web demo note

The browser viewer can load the avatar and interface without a bridge, but it will not reach your AI backend until you pair it to her-bridge with the tunnel URL.

Apple device note

iOS, iPad, and iWatch are still in testing and are not part of the public demo yet.
